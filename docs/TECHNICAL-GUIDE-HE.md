## 🎲 מערכת Events — איך ההסתברות עובדת בדיוק?

### התשובה הקצרה
ההסתברות מבוססת על **זמן (שניות)**, לא על כמות אריחים שנזרקו.

### התשובה המפורטת

המערכת בודקת **כל שנייה** (1,000 מילישניות) האם להציג event חדש. הבדיקה עוברת **7 שלבים**:

```
כל שנייה:
│
├─ שלב 1: האם events מופעלים? (events_enabled = true?)
│          אם לא → לא קורה כלום
│
├─ שלב 2: האם יש כבר event פעיל על הלוח?
│          אם כן → מחכה שייגמר (לא 2 events בו-זמנית)
│
├─ שלב 3: האם עבר מספיק זמן מתחילת המשחק?
│          צריך לעבור events_start_delay שניות (ברירת מחדל: 15)
│          אם לא → מחכה
│
├─ שלב 4: האם עבר מספיק זמן מה-event האחרון?
│          צריך לעבור events_min_gap שניות (ברירת מחדל: 15)
│          אם לא → מחכה
│
├─ שלב 5: האם יש מספיק תאים ריקים?
│          צריך events_min_empty_cells תאים (ברירת מחדל: 4)
│          אם לא → לא מופיע (הלוח כמעט מלא)
│
├─ שלב 6: הגרלת הסתברות (הלב של המערכת!)
│          ← ראה פירוט למטה
│
└─ שלב 7: אם עבר הגרלה → בחירת סוג event + מיקום
```

### שלב 6 — הגרלת ההסתברות (איך זה עובד בדיוק)

ההסתברות **עולה בהדרגה** ככל שעובר יותר זמן מה-event האחרון:

```
זמן שעבר מ-event אחרון:
0-15 שניות  → הסתברות = 0% (ממתין, min_gap)
15 שניות    → הסתברות = 0% (הגרלה מתחילה)
20 שניות    → הסתברות = ~10% לשנייה
25 שניות    → הסתברות = ~20% לשנייה
30 שניות    → הסתברות = ~30% לשנייה
35 שניות    → הסתברות = 40% לשנייה (מקסימום)
35+ שניות   → נשאר 40% לשנייה עד שקורה
```

**הנוסחה המדויקת:**
```
elapsed = שניות שעברו מאז event אחרון
prob = (elapsed - min_gap) / (max_gap - min_gap)    ← ערך בין 0 ל-1
prob = min(prob, 1)                                   ← מוגבל ל-100%
chance_per_second = prob × 0.4                        ← מכפיל 40%

כל שנייה: random() < chance_per_second?
  כן → event מופיע!
  לא → בודק שוב בעוד שנייה
```

**דוגמה מספרית (הגדרות ברירת מחדל: min=15, max=35):**
```
שנייה 0:  המשחק מתחיל
שנייה 15: events_start_delay עבר, מתחילים לבדוק
שנייה 15: min_gap לא עבר (0 שניות מ-event אחרון) → מחכה
שנייה 18: event ראשון מופיע (force spawn אחרי 3 שניות)
שנייה 18: event על הלוח, טיימר 8 שניות
שנייה 26: event נגמר (נלחץ או פג תוקף)
שנייה 26: lastEventTime = עכשיו
שנייה 26-41: min_gap (15 שניות) → מחכה
שנייה 41: prob = (0)/(20) = 0% → סיכוי 0%
שנייה 46: prob = (5)/(20) = 25% → סיכוי 10% לשנייה
שנייה 51: prob = (10)/(20) = 50% → סיכוי 20% לשנייה
שנייה 56: prob = (15)/(20) = 75% → סיכוי 30% לשנייה
שנייה 61: prob = (20)/(20) = 100% → סיכוי 40% לשנייה

בממוצע: event חדש מופיע כל ~25-35 שניות
```

### מה האדמין יכול לשנות?

| רוצה | מה לשנות | דוגמה |
|------|----------|-------|
| events תכופים יותר | הקטן `events_min_gap` | `events_min_gap` = `8` |
| events נדירים | הגדל `events_min_gap` + `events_max_gap` | `min=60`, `max=120` |
| event מיידי בתחילת משחק | הקטן `events_start_delay` | `events_start_delay` = `5` |
| events רק אחרי 2 דקות | הגדל `events_start_delay` | `events_start_delay` = `120` |
| events רק כשהלוח כמעט ריק | הגדל `events_min_empty_cells` | `events_min_empty_cells` = `12` |
| כבה events לגמרי | `events_enabled` = `false` | — |

## 🎰 בחירת סוג Event — Weighted Random

אחרי שהמערכת החליטה שevent יופיע, היא מגרילה **איזה סוג**:

### איך הגרלת Weight עובדת

```
כל event יש "משקל" (weight).
המערכת סוכמת את כל המשקלים ומגרילה מספר.

דוגמה עם ברירת מחדל:
💣 bomb   = 25
⭐ star   = 20
🎁 gift   = 25
🔥 fever  = 12
❄️ freeze = 8  (רק אם ≥3 שורות מלאות!)
🎯 target = 10
─────────────
סה"כ = 100

הגרלה: מספר רנדומלי 0-100
0-25   → 💣 פצצה
25-45  → ⭐ כוכב
45-70  → 🎁 מתנה
70-82  → 🔥 Fever
82-90  → ❄️ הקפאה
90-100 → 🎯 מטרה
```

### מקרה מיוחד: ❄️ Freeze

Freeze **לא נכנס להגרלה** אם הלוח לא מספיק מלא:
```
אם שורות מלאות < event_freeze_min_filled_rows (ברירת מחדל: 3)
  → Freeze לא בהגרלה
  → סה"כ weights = 92 (בלי 8 של freeze)
  → bomb = 25/92 = 27%
  → star = 20/92 = 22%
  → ...
```

### דוגמאות שינוי הסתברויות

**"רוצה הרבה פצצות":**
```
event_bomb_weight = 50 (היה 25)
סה"כ = 125
bomb = 50/125 = 40% (היה 25%)
```

**"רוצה רק מתנות ופצצות":**
```
event_bomb_weight = 50
event_gift_weight = 50
event_star_weight = 0
event_fever_weight = 0
event_freeze_weight = 0
event_target_weight = 0
סה"כ = 100
bomb = 50% · gift = 50%
```

**"רוצה Fever כל הזמן":**
```
event_fever_weight = 80
כל השאר = 5
סה"כ = 105
fever = 80/105 = 76%
```

## ⏱️ מה קורה אחרי שEvent מופיע?

```
Event מופיע על הלוח
│
├─ טיימר מתחיל (event_X_timer שניות)
│  ירוק (>50%) → צהוב (25-50%) → אדום מהבהב (<25%)
│
├─ שחקן מפיל אריח לעמודה של ה-event?
│  ├─ כן → EVENT TRIGGERED! (אפקט + בונוס)
│  └─ לא → ממשיך לספור אחורה
│
├─ טיימר הגיע ל-0?
│  └─ כן → Event נעלם (הזדמנות הוחמצה)
│
└─ אחרי trigger או timeout:
   └─ lastEventTime = עכשיו
   └─ מחכה min_gap שניות עד event הבא
```

### טיימרים לכל סוג

| Event | טיימר ברירת מחדל | מפתח |
|-------|------------------|------|
| 💣 פצצה | 8 שניות | `event_bomb_timer` |
| ⭐ כוכב | 6 שניות | `event_star_timer` |
| 🎁 מתנה | 10 שניות | `event_gift_timer` |
| 🔥 Fever | 5 שניות | `event_fever_timer` |
| ❄️ הקפאה | 4 שניות (דחוף!) | `event_freeze_timer` |
| 🎯 מטרה | 12 שניות (ארוך) | `event_target_timer` |

## 💣 פירוט כל Event — מה בדיוק קורה

### 💣 פצצה (Bomb)
```
מופיע: תא ריק בתחתית עמודה רנדומלית
טיימר: 8 שניות (event_bomb_timer)
הפעלה: שחקן מפיל אריח לעמודה

מה קורה:
1. כל האריחים בטווח radius נמחקים מהלוח
   - radius=1: 4 שכנים (למעלה, למטה, ימין, שמאל)
   - radius=2: 12 תאים (ריבוע 5×5 בלי פינות)
2. בונוס: (מספר אריחים שנמחקו) × event_bomb_points_per_tile
   - ברירת מחדל: 4 × 2,000 = 8,000 נקודות
3. gravity — אריחים מעל נופלים למטה
4. Screen shake (event_bomb_shake = 6)
5. באנר "💣 BOOM! +8,000"
6. ויברציה

הגדרות:
- event_bomb_radius = 1 (ברירת מחדל)
- event_bomb_points_per_tile = 2000
- event_bomb_shake = 6
```

### ⭐ כוכב זהב (Star)
```
מופיע: תא ריק בתחתית עמודה רנדומלית
טיימר: 6 שניות
הפעלה: שחקן מפיל אריח לעמודה

מה קורה:
1. האריח שנחת מקפיץ דרגה (עלה→פרח, יהלום→כתר!)
   - מספר דרגות: event_star_upgrade (ברירת מחדל: 1)
2. +500 נקודות (event_star_points)
3. אם האריח כבר כתר → +2,500 נקודות
4. הלוח מתעדכן מיד

למה זה ממכר:
- שחקן "שומר" אריח גבוה לעמודה עם הכוכב
- יהלום + כוכב = כתר בחינם!
```

### 🎁 מתנה (Gift)
```
מופיע: תא ריק בתחתית עמודה רנדומלית
טיימר: 10 שניות (ארוך — פחות לחץ)
הפעלה: שחקן מפיל אריח לעמודה

מה קורה:
1. הגרלת פרס:
   - 95% סיכוי: רנדומלי בין event_gift_credits_min ל-event_gift_credits_max
     (ברירת מחדל: 5-50 💎)
   - 5% סיכוי: JACKPOT! event_gift_jackpot_amount
     (ברירת מחדל: 500 💎)
2. קרדיטים נוספים לחשבון השחקן (בשרת!)
3. באנר "🎁 מתנה! +23 💎" או "🎁 JACKPOT!!! +500 💎"
4. Rate limit: מקסימום gift אחד כל 10 שניות

הגדרות:
- event_gift_credits_min = 5
- event_gift_credits_max = 50
- event_gift_jackpot_chance = 5 (אחוז)
- event_gift_jackpot_amount = 500
```

### 🔥 Fever Mode
```
מופיע: תא ריק בתחתית עמודה רנדומלית
טיימר: 5 שניות (להפעלה)
הפעלה: שחקן מפיל אריח לעמודה

מה קורה:
1. Fever Mode מופעל למשך event_fever_duration שניות (10)
2. כל מיזוג מקבל × event_fever_multiplier ניקוד (×3)
3. גבול הלוח בוער (אפקט אש)
4. בר countdown בראש המסך
5. "🔥 ×3" מהבהב
6. אחרי 10 שניות — חוזר לנורמלי

דוגמה:
- מיזוג רגיל של כוכב = 300 נקודות
- מיזוג כוכב ב-Fever = 300 × 3 = 900 נקודות!
- שרשרת ×2 + Fever = 300 × 2 × 3 = 1,800 נקודות!

הגדרות:
- event_fever_duration = 10 (שניות)
- event_fever_multiplier = 3
```

### ❄️ הקפאה (Freeze)
```
מופיע: רק כשהלוח כמעט מלא (≥3 שורות מלאות)
טיימר: 4 שניות (קצר — דחוף!)
הפעלה: שחקן מפיל אריח לעמודה

תנאי מיוחד: Freeze לא יופיע אם הלוח לא מספיק מלא!
  → event_freeze_min_filled_rows = 3

מה קורה:
1. שורה עליונה נמחקת לגמרי (event_freeze_clear_rows = 1)
2. +1,000 נקודות (event_freeze_points)
3. אפקט קרח כחול על השורה
4. gravity — אריחים נופלים למטה
5. Screen shake

למה זה ממכר:
- מופיע ברגע שהשחקן חושב "הולך להיגמר!"
- הצלת חיים ברגע האחרון
- מאריך את המשחק → ניקוד גבוה יותר

הגדרות:
- event_freeze_clear_rows = 1
- event_freeze_points = 1000
- event_freeze_min_filled_rows = 3
- event_freeze_shake = 4
```

### 🎯 מטרה (Target)
```
שונה מכל השאר! לא מופיע על תא — מופיע על ה-TIER BAR

טיימר: 12 שניות (ארוך — צריך לתכנן)
הפעלה: שחקן ממזג 2 אריחים מהדרגה המסומנת

מה קורה:
1. דרגה רנדומלית (2-6) מקבלת מסגרת סגולה מהבהבת
2. שחקן צריך למזג את הדרגה הזו תוך 12 שניות
3. אם הצליח: המיזוג מקבל × event_target_multiplier ניקוד (×5!)
4. אם לא: הסימון נעלם

דוגמה:
- Target על "פרח" (דרגה 3)
- מיזוג רגיל של פרח = 96 נקודות
- מיזוג פרח עם Target = 96 × 5 = 480 נקודות!

הגדרות:
- event_target_multiplier = 5
- event_target_timer = 12
```

## 📳 מערכת Screen Shake

כשאירוע דרמטי קורה, הלוח רועד. **כל רעידה ניתנת לשליטה (0 = כבוי).**

### איך זה עובד
```
shakeGrid(intensity):
1. לולאה של 6 מיקרו-תנועות (כל 40 מילישניות)
2. כל תנועה: הלוח זז רנדומלית ±intensity פיקסלים
3. אחרי 6 תנועות: חוזר למקום

intensity=2: רעידה עדינה (בקושי מורגשת)
intensity=4: רעידה בינונית (מרגישים אותה)
intensity=6: רעידה חזקה (דרמטית)
intensity=8: רעידת אדמה! (Crown Merge)
intensity=10+: מוגזם (למטרות מיוחדות)
```

### כל אירועי Shake

| אירוע | עוצמה | מפתח | מתי קורה |
|-------|--------|------|----------|
| מיזוג tier 5 (ברק) | 3 | `shake_tier_up` | פעם ראשונה במשחק |
| מיזוג tier 6 (כוכב) | 4 | `shake_tier_up` | פעם ראשונה במשחק |
| מיזוג tier 7 (יהלום) | 5 | `shake_tier_up` | פעם ראשונה במשחק |
| Crown Merge | 8 | `shake_crown_merge` | כל פעם שמזגים 2 כתרים |
| Score milestone | 2 | `shake_milestone` | 10K, 25K, 50K, 100K, 200K |
| Triple merge (3 אריחים) | 3 | `shake_multi_merge` | כל triple |
| Quad merge (4 אריחים) | 6 | `shake_multi_merge` | כל quad |
| שיא חדש | 4 | `shake_new_best` | ברגע שעוברים את השיא |
| 💣 Bomb event | 6 | `event_bomb_shake` | כל פיצוץ |
| ❄️ Freeze event | 4 | `event_freeze_shake` | כל הקפאה |

## 🔗 שרשראות (Chains)

```
כשמיזוג גורם לאריח חדש → שנופל → שמתמזג שוב

שרשרת ×1: מיזוג רגיל (multiplier = 1.0)
שרשרת ×2: מיזוג → נפילה → מיזוג שני (multiplier = 1.5)
שרשרת ×3: שלושה מיזוגים ברצף (multiplier = 2.0)
שרשרת ×4: ארבעה מיזוגים ברצף (multiplier = 2.5)

נוסחה: multiplier = 1 + (chainCount - 1) × 0.5

עם Fever:
שרשרת ×3 + Fever ×3 = ניקוד × 2.0 × 3.0 = ×6!

עם Target:
מיזוג דרגה מסומנת + Target ×5 = ניקוד × 5.0!

עם Target + Fever:
ניקוד × 5.0 × 3.0 = ×15!
```

## 🧠 Triple / Quad Merge

```
כשמיזוג כולל 3+ אריחים בו-זמנית (לא chain, אלא קבוצה):

3 אריחים → "✨ Triple!" → shake(3)
4 אריחים → "💥 Quad!"  → shake(6)
5+ אריחים → "🌟 MEGA!"  → shake(6)

הניקוד כבר מחושב לפי groupSize בנוסחה:
points = tier × 10 × (1 + (tier-1) × 0.3) × groupSize × chainMult

אז Triple של פרח:
  3 × 10 × (1 + 2×0.3) × 3 × 1 = 3 × 10 × 1.6 × 3 = 144 נקודות
  (במקום 96 של pair)
```

## 🎯 Game Over — פסיכולוגיית ה-Retry
